Chapter 1: Introduction to Roblox and Lua

Welcome to the world of Roblox game development. In this chapter, you will discover what makes Roblox a unique platform for creators and why Lua is the perfect language for bringing your game ideas to life.

1.1 What is Roblox?

Roblox is more than just a gaming platform; it is a global ecosystem where millions of players become creators. Launched in 2006, Roblox has grown into one of the largest user-generated content platforms in the world, with over 70 million daily active users and millions of experiences created by its community.

At its core, Roblox consists of three interconnected components:

  • Roblox Client: The application players use to join and play experiences
  • Roblox Studio: The powerful, free development environment where creators build games
  • Roblox Cloud: The infrastructure that hosts and delivers experiences globally

What sets Roblox apart from traditional game development is its accessibility. While creating a game from scratch typically requires extensive programming knowledge, expensive software, and complex deployment processes, Roblox provides an integrated environment where anyone can start building immediately.

1.2 Why Learn Roblox Lua?

Lua is the scripting language that powers Roblox experiences. But why was Lua chosen, and what makes it ideal for game development?

Lightweight and Fast: Lua was designed to be embeddable and efficient. Its small footprint and high performance make it perfect for real-time applications like games where every millisecond counts.

Easy to Learn: Lua's syntax is clean and readable. If you have never programmed before, Lua provides a gentle learning curve. If you know other languages, you will appreciate its elegant simplicity.

Powerful and Flexible: Despite its simplicity, Lua is a fully capable programming language. It supports multiple programming paradigms including procedural, object-oriented, and functional programming.

Industry Recognition: Learning Lua opens doors beyond Roblox. It is used in Adobe products, World of Warcraft, Angry Birds, and many other professional applications.

Did You Know?

Lua means "moon" in Portuguese. It was created in 1993 at the Pontifical Catholic University of Rio de Janeiro, Brazil, by Roberto Ierusalimschy, Luiz Henrique de Figueiredo, and Waldemar Celes.

1.3 How Roblox Lua Works

When you write Lua code in Roblox Studio, you are creating scripts that control the behavior of objects in your game world. These scripts run within the Roblox engine, which handles rendering, physics, networking, and many other complex tasks.

Roblox uses a modified version of Lua 5.1 called Luau, which adds type inference, performance improvements, and additional features while maintaining compatibility with standard Lua.

Scripts in Roblox can run in two contexts:

  • Server Scripts: Run on Roblox servers, handling game logic, data storage, and authoritative decisions
  • Local Scripts: Run on each player's device, handling user input, camera control, and visual effects

Chapter 3: Lua Fundamentals - Variables and Data Types

Variables and data types are the building blocks of programming. This chapter covers everything you need to know about storing and working with information in Lua.

3.1 What Are Variables?

A variable is a named container that stores a value. Think of it as a labeled box where you can put data and retrieve it later by name. In Lua, you create variables using the local keyword:

-- Creating variables
local playerName = "Alex"
local playerScore = 1000
local isVIP = true

-- Using variables
print(playerName)        -- Output: Alex
print(playerScore)       -- Output: 1000
print(isVIP)             -- Output: true

The local keyword creates a variable with limited scope, meaning it is only accessible within its defined region. This is the recommended practice in Roblox as it improves performance and prevents naming conflicts.

3.2 Lua Data Types

Lua has eight basic data types. Understanding each is essential for effective programming:

Nil

Nil represents the absence of a value. It is the default value for uninitialized variables.

local emptyVariable
print(emptyVariable)  -- Output: nil

-- You can explicitly set something to nil
local temporaryData = "some data"
temporaryData = nil  -- Now eligible for garbage collection

Boolean

Booleans represent true or false values, essential for decision-making in code.

local hasKey = true
local doorLocked = false

print(hasKey)      -- Output: true
print(doorLocked)  -- Output: false

Number

Lua uses double-precision floating-point numbers for all numeric values. This includes integers, decimals, and scientific notation.

local integer = 42
local decimal = 3.14159
local negative = -100
local scientific = 1.5e10  -- 15,000,000,000

print(integer + decimal)  -- Output: 45.14159

String

Strings are sequences of characters used for text. They can be enclosed in single or double quotes.

local singleQuotes = 'Hello'
local doubleQuotes = "World"
local multiline = [[
    This is a
    multi-line string
]]

-- String concatenation uses ..
local greeting = singleQuotes .. " " .. doubleQuotes
print(greeting)  -- Output: Hello World

Table

Tables are Lua's universal data structure. They can function as arrays, dictionaries, objects, or combinations thereof. We will explore tables in depth in Chapter 8.

local inventory = {"sword", "shield", "potion"}
local playerData = {
    name = "Alex",
    level = 25,
    health = 100
}

Function

Functions are first-class values in Lua, meaning they can be stored in variables, passed as arguments, and returned from other functions.

local function greet(name)
    return "Hello, " .. name .. "!"
end

print(greet("Player"))  -- Output: Hello, Player!

Userdata

Userdata represents arbitrary C data. In Roblox, this is primarily used for Roblox-specific objects like Parts, Players, and Services.

-- workspace is a userdata representing the Workspace service
local part = workspace.Part
print(typeof(part))  -- Output: Instance

Thread

Threads represent independent lines of execution. In Roblox, these are commonly used through the task library for asynchronous operations.

-- Creating a coroutine (cooperative thread)
local co = coroutine.create(function()
    print("Running in coroutine")
end)

coroutine.resume(co)

3.3 Variable Naming Rules

When naming variables, follow these rules:

  • Names can contain letters, numbers, and underscores
  • Names must begin with a letter or underscore
  • Names are case-sensitive (Score and score are different)
  • Avoid Lua reserved words: and, break, do, else, elseif, end, false, for, function, if, in, local, nil, not, or, repeat, return, then, true, until, while
Naming Conventions

Use camelCase for variables (playerName, maxHealth). Use PascalCase for functions that create objects or modules. Use UPPER_SNAKE_CASE for constants (MAX_PLAYERS, DEFAULT_SPEED).

3.4 Type Checking and Conversion

You can check a variable's type using the typeof() function (Roblox) or type() function (standard Lua):

local value = 42
print(typeof(value))  -- Output: number

value = "hello"
print(typeof(value))  -- Output: string

Converting between types:

-- Number to string
local num = 42
local str = tostring(num)  -- "42"

-- String to number
local str = "3.14"
local num = tonumber(str)  -- 3.14

-- String to number (invalid returns nil)
local invalid = tonumber("abc")  -- nil

3.5 Constants in Roblox

While Lua does not have true constants, Roblox provides the const keyword in Luau for values that should not change:

local MAX_HEALTH = 100
local GAME_VERSION = "1.0.0"
local PI = 3.14159

-- Convention: uppercase with underscores indicates intent

  • Variables store data using the local keyword for proper scoping
  • Lua has eight data types: nil, boolean, number, string, table, function, userdata, thread
  • Strings can use single quotes, double quotes, or double brackets for multiline

Logical operators combine boolean expressions:

4.4 String Operators

The concatenation operator joins strings:

-- Concatenation with ..

local lastName = "Doe"

-- Concatenating numbers (automatically converted)
local score = 1000
local message = "Your score: " .. score  -- "Your score: 1000"

-- Length operator #
local text = "Hello"
local length = #text  -- 5

-- String repetition (Luau feature)
local line = "-"
local separator = line:rep(20)  -- "--------------------"

4.5 Operator Precedence

Operators are evaluated in a specific order. Use parentheses to control evaluation:

-- Without parentheses
local result = 5 + 3 * 2  -- 11 (multiplication first)

-- With parentheses
local result = (5 + 3) * 2  -- 16

-- Complex expression
local health = 100
local damage = 25
local armor = 10
local finalDamage = damage - (armor * 0.5)  -- 20

4.6 Compound Assignment Operators

Luau supports compound assignment operators for cleaner code:

local score = 100

-- Traditional way
score = score + 50

-- Compound assignment (Luau)
score += 50  -- Equivalent to score = score + 50
score -= 10  -- Subtract
score *= 2   -- Multiply
score /= 2   -- Divide

Chapter 5: Control Flow - Making Decisions

Programs need to make decisions based on conditions. This chapter explores Lua's control flow statements that enable your code to respond dynamically to different situations.

5.1 The if Statement

The if statement executes code only when a condition is true:

Syntax:
if condition then
    -- code to execute
end 
local playerHealth = 25
local maxHealth = 100

-- Simple if statement
if playerHealth < maxHealth * 0.3 then
    print("Warning: Low health!")
end

5.2 The if-else Statement

Use else to provide an alternative when the condition is false:

Syntax:
if condition then
    -- code when true
else
    -- code when false
end 
local hasPermission = false

if hasPermission then
    print("Access granted")
else
    print("Access denied")
end

5.3 The if-elseif-else Statement

Chain multiple conditions with elseif:

Syntax:
if condition1 then
    -- code for condition1
elseif condition2 then
    -- code for condition2
elseif condition3 then
    -- code for condition3
else
    -- default code
end 
local playerLevel = 45

if playerLevel < 10 then
    print("Beginner rank")
elseif playerLevel < 30 then
    print("Intermediate rank")
elseif playerLevel < 60 then
    print("Advanced rank")
else
    print("Expert rank")
end

5.4 Nested if Statements

You can place if statements inside other if statements:

local isLoggedIn = true
local accountAge = 30
local hasVerifiedEmail = true

if isLoggedIn then
    if accountAge >= 7 then
        if hasVerifiedEmail then
            print("You can trade items")
        else
            print("Please verify your email to trade")
        end
    else
        print("Account must be at least 7 days old to trade")
    end
else
    print("Please log in to trade")
end

5.5 Logical Operators in Conditions

Combine conditions with and, or, and not:

local level = 50
local hasQuestItem = true
local questCompleted = false

-- Using AND: both conditions must be true
if level >= 40 and hasQuestItem then
    print("You can enter the dungeon")
end

-- Using OR: at least one condition must be true
if level >= 60 or questCompleted then
    print("You have VIP access")
end

-- Combining operators
local isAdmin = false
local isModerator = true
local isBanned = false

if (isAdmin or isModerator) and not isBanned then
    print("Moderation tools available")
end

5.6 Ternary-like Expressions

Lua does not have a traditional ternary operator, but you can achieve similar results:

local score = 85

-- Traditional if-else
local grade
if score >= 60 then
    grade = "Pass"
else
    grade = "Fail"
end

-- Using and/or pattern (ternary-like)
local grade = score >= 60 and "Pass" or "Fail"

-- Another example
local playerName = nil
local displayName = playerName or "Guest"  -- "Guest" if playerName is nil
Caution with Ternary Pattern

The and/or pattern only works reliably when the "true" value is not false or nil. For complex cases, use explicit if-else statements for clarity.

5.7 Practical Examples in Roblox

-- Check if player can afford an item
local playerMoney = 500
local itemPrice = 300

if playerMoney >= itemPrice then
    playerMoney = playerMoney - itemPrice
    print("Purchase successful! Remaining: " .. playerMoney)
else
    local needed = itemPrice - playerMoney
    print("Insufficient funds. Need " .. needed .. " more")
end

-- Validate player input
local username = "Player123"

if #username < 3 then
    print("Username too short")
elseif #username > 20 then
    print("Username too long")
elseif username:find(" ") then
    print("Username cannot contain spaces")
else
    print("Username valid!")
end

Chapter 6: Loops and Iteration

Loops allow code to repeat, which is essential for processing collections, creating animations, and implementing game mechanics. This chapter covers all loop types in Lua.

6.1 The while Loop

A while loop repeats as long as its condition remains true:

Syntax:
while condition do
    -- code to repeat
end 
-- Countdown timer
local countdown = 10

while countdown > 0 do
    print("Time remaining: " .. countdown)
    countdown = countdown - 1
    task.wait(1)  -- Wait 1 second (Roblox function)
end

print("Go!")

-- Health regeneration
local health = 50
local maxHealth = 100

while health < maxHealth do
    health = health + 5
    print("Health: " .. health)
    task.wait(0.5)
end
Infinite Loops

Ensure your while loop condition eventually becomes false. In Roblox, infinite loops without yielding (task.wait) will freeze the game.

6.2 The repeat-until Loop

A repeat loop executes at least once, then continues until its condition becomes true:

Syntax:
repeat
    -- code to repeat
until condition 
-- Password validation (must try at least once)
local correctPassword = "secret123"
local attempts = 0
local input

repeat
    attempts = attempts + 1
    input = "secret123"  -- Simulated input
    print("Attempt " .. attempts)
until input == correctPassword or attempts >= 3

print(attempts < 3 and "Access granted!" or "Account locked!")

-- Dice roll until we get a 6
local rolls = 0
local roll

repeat
    roll = math.random(1, 6)
    rolls = rolls + 1
    print("Rolled: " .. roll)
until roll == 6

print("Got 6 in " .. rolls .. " rolls!")

6.3 The for Loop (Numeric)

The numeric for loop iterates a specific number of times:

Syntax:
for variable = start, finish, step do
    -- code to repeat
end 
-- Basic counting
for i = 1, 5 do
    print("Iteration " .. i)
end

-- With custom step (count by 2s)
for i = 0, 10, 2 do
    print(i)  -- 0, 2, 4, 6, 8, 10
end

-- Counting backwards
for i = 10, 1, -1 do
    print(i)  -- 10, 9, 8, ... 1
end

-- Creating a grid
for row = 1, 3 do
    for col = 1, 3 do
        print("Cell (" .. row .. ", " .. col .. ")")
    end
end

6.4 The for Loop (Generic)

The generic for loop iterates over collections using iterators:

-- Iterating over an array
local fruits = {"apple", "banana", "cherry"}

for index, value in ipairs(fruits) do
    print(index .. ": " .. value)
end

-- Iterating over a dictionary
local playerData = {
    name = "Alex",
    level = 25,
    health = 100
}

for key, value in pairs(playerData) do
    print(key .. " = " .. tostring(value))
end

-- Using string.gmatch for text processing
local text = "apple, banana, cherry"
for fruit in text:gmatch("[^, ]+") do
    print(fruit)
end
ipairs vs pairs

Use ipairs for arrays (ordered, numeric indices starting at 1). Use pairs for dictionaries (unordered, any key type). ipairs stops at the first nil, while pairs iterates all non-nil keys.

6.5 Loop Control: break and continue

Control loop execution flow:

-- break: exit the loop immediately
for i = 1, 100 do
    if i > 10 then
        break  -- Stop after 10
    end
    print(i)
end

-- continue: skip to next iteration (Luau)
for i = 1, 10 do
    if i % 2 == 0 then
        continue  -- Skip even numbers
    end
    print(i)  -- Only odd numbers: 1, 3, 5, 7, 9
end

-- Finding a specific player
local players = {"Alice", "Bob", "Charlie", "Diana"}
local target = "Charlie"

for i, name in ipairs(players) do
    if name == target then
        print("Found " .. target .. " at position " .. i)
        break
    end
end

6.6 Practical Loop Applications

-- Spawn multiple enemies
for i = 1, 5 do
    local enemy = Instance.new("Part")
    enemy.Name = "Enemy" .. i
    enemy.Position = Vector3.new(i * 10, 5, 0)
    enemy.Parent = workspace
end

-- Calculate average score
local scores = {85, 92, 78, 95, 88}

Chapter 9: Understanding OOP in Lua

Object-oriented programming helps organize code into reusable, modular components. While Lua does not have built-in classes, it provides powerful mechanisms for OOP.

9.1 Prototype-Based OOP

Lua uses prototypes rather than classes. An object is simply a table that can inherit from another table:

-- Create a prototype (like a class)
local Animal = {}
Animal.__index = Animal

-- Constructor
function Animal.new(name)
    local self = setmetatable({}, Animal)
    self.name = name
    self.health = 100
    return self
end

-- Methods
function Animal:speak()
    print(self.name .. " makes a sound")
end

function Animal:takeDamage(amount)
    self.health = self.health - amount
    print(self.name .. " health: " .. self.health)
end

-- Create instances
local dog = Animal.new("Buddy")
dog:speak()        -- Buddy makes a sound
dog:takeDamage(10) -- Buddy health: 90

9.2 The self Parameter

The colon (:) syntax automatically passes the object as the first parameter:

-- These are equivalent:
function Animal:speak() end
function Animal.speak(self) end

-- Calling methods:
animal:speak()      -- Passes animal as self
animal.speak(animal) -- Explicit self

9.3 Inheritance

Create child prototypes that inherit from parents:

-- Dog inherits from Animal
local Dog = setmetatable({}, Animal)
Dog.__index = Dog

function Dog.new(name, breed)
    local self = setmetatable(Animal.new(name), Dog)
    self.breed = breed
    return self
end

-- Override parent method
function Dog:speak()
    print(self.name .. " barks!")
end

-- New method specific to Dog
function Dog:fetch()
    print(self.name .. " fetches the ball!")
end

-- Usage
local myDog = Dog.new("Rex", "German Shepherd")
myDog:speak()      -- Rex barks! (overridden)
myDog:fetch()      -- Rex fetches the ball!
myDog:takeDamage(5) -- Inherited from Animal

9.4 Multiple Inheritance

Lua supports multiple inheritance through careful metatable setup:

-- Create a mixin for flying ability
local Flyable = {}

function Flyable:fly()
    print(self.name .. " is flying!")
end

function Flyable:land()
    print(self.name .. " lands gently.")
end

-- Bird inherits from Animal and Flyable
local Bird = setmetatable({}, Animal)
Bird.__index = function(table, key)
    return Bird[key] or Flyable[key] or Animal[key]
end

function Bird.new(name, wingSpan)
    local self = setmetatable(Animal.new(name), Bird)
    self.wingSpan = wingSpan
    return self
end

function Bird:speak()
    print(self.name .. " chirps!")
end

-- Usage
local eagle = Bird.new("Eddie", 2.5)
eagle:speak()   -- Eddie chirps!
eagle:fly()     -- Eddie is flying! (from Flyable)

9.5 Private Members

Implement privacy using closures:

local function createBankAccount(owner, initialBalance)
    -- Private variables (closure)
    local balance = initialBalance or 0
    local transactionHistory = {}
    
    -- Public interface
    local account = {}
    account.owner = owner
    
    function account:deposit(amount)
        if amount > 0 then
            balance = balance + amount
            table.insert(transactionHistory, {"deposit", amount})
            return true
        end
        return false
    end
    
    function account:withdraw(amount)
        if amount > 0 and amount <= balance then
            balance = balance - amount
            table.insert(transactionHistory, {"withdraw", amount})
            return true
        end
        return false
    end
    
    function account:getBalance()
        return balance
    end
    
    function account:getHistory()
        -- Return a copy to prevent modification
        local copy = {}
        for i, v in ipairs(transactionHistory) do
            copy[i] = {v[1], v[2]}
        end
        return copy
    end
    
    return account
end

-- Usage
local account = createBankAccount("Alex", 100)
account:deposit(50)
account:withdraw(30)
print(account:getBalance())  -- 120
-- print(account.balance)    -- nil (private)
Key Takeaways
  • Lua uses prototype-based OOP with tables and metatables
  • Use __index for inheritance
  • The colon (:) syntax provides automatic self parameter
  • Override parent methods by defining them in the child
  • Use closures to create truly private members

Chapter 10: Metatables and Metamethods

Metatables are one of Lua's most powerful features. They allow you to change the behavior of tables, enabling operator overloading and advanced patterns.

10.1 What Are Metatables?

A metatable is a regular table that defines special behaviors for another table. You attach it using setmetatable:

local myTable = {}
local myMetatable = {}

setmetatable(myTable, myMetatable)

-- Or create both at once
local t = setmetatable({}, {
    __index = function(table, key)
        return "Key " .. key .. " not found"
    end
})

print(t.someKey)  -- "Key someKey not found"

10.2 The __index Metamethod

__index is called when you access a missing key:

-- Default values for missing keys
local defaults = {
    health = 100,
    mana = 50,
    speed = 10
}

local player = setmetatable({name = "Alex"}, {
    __index = defaults
})

print(player.name)    -- Alex (from player)
print(player.health)  -- 100 (from defaults)
print(player.mana)    -- 50 (from defaults)

10.3 The __newindex Metamethod

__newindex is called when you assign to a missing key:

-- Read-only table
local function createReadOnlyTable(original)
    return setmetatable({}, {
        __index = original,
        __newindex = function()
            error("Attempt to modify read-only table")
        end
    })
end

local constants = createReadOnlyTable({
    PI = 3.14159,
    MAX_PLAYERS = 20
})

print(constants.PI)  -- 3.14159
-- constants.PI = 3  -- Error!

10.4 Arithmetic Metamethods

Overload arithmetic operators for custom behavior:

-- Vector2 class with operator overloading
local Vector2 = {}
Vector2.__index = Vector2

function Vector2.new(x, y)
    return setmetatable({x = x or 0, y = y or 0}, Vector2)
end

-- Addition
function Vector2.__add(a, b)
    return Vector2.new(a.x + b.x, a.y + b.y)
end

-- Subtraction
function Vector2.__sub(a, b)
    return Vector2.new(a.x - b.x, a.y - b.y)
end

-- Multiplication (scalar or vector)
function Vector2.__mul(a, b)
    if type(a) == "number" then
        return Vector2.new(b.x * a, b.y * a)
    elseif type(b) == "number" then
        return Vector2.new(a.x * b, a.y * b)
    else
        return Vector2.new(a.x * b.x, a.y * b.y)
    end
end

-- String representation
function Vector2.__tostring(v)
    return "Vector2(" .. v.x .. ", " .. v.y .. ")"
end

-- Usage
local v1 = Vector2.new(3, 4)
local v2 = Vector2.new(1, 2)

print(v1 + v2)      -- Vector2(4, 6)
print(v1 - v2)      -- Vector2(2, 2)
print(v1 * 2)       -- Vector2(6, 8)
print(v1 * v2)      -- Vector2(3, 8)

10.5 Comparison Metamethods

Define how objects compare to each other:

-- Equality and ordering
function Vector2.__eq(a, b)
    return a.x == b.x and a.y == b.y
end

function Vector2.__lt(a, b)
    return a.x < b.x or (a.x == b.x and a.y < b.y)
end

function Vector2.__le(a, b)
    return a < b or a == b
end

-- Usage
local v1 = Vector2.new(3, 4)
local v2 = Vector2.new(3, 4)
local v3 = Vector2.new(5, 6)

print(v1 == v3)  -- false
print(v1 < v3)   -- true

Chapter 11: Building Reusable Modules

Modules are the foundation of organized Roblox code. This chapter teaches you to create, structure, and use modules effectively.

11.1 What Are Modules?

Modules are scripts that return a table of functions and data, designed to be reused across your game:

-- In a ModuleScript named "MathUtils"

    return math.max(min, math.min(max, value))
end

function MathUtils.lerp(a, b, t)
    return a + (b - a) * t
end

function MathUtils.round(num, decimals)
    local mult = 10 ^ (decimals or 0)
    return math.floor(num * mult + 0.5) / mult
end

return MathUtils

-- Second require: returns cached result

print(Utils1 == Utils2)  -- true (same table)

-- This enables shared state

print(Utils2.counter)  -- 5
Key Takeaways
  • Modules return tables of reusable code
  • Use require() to load modules
  • Modules are cached after first require
  • Organize modules by purpose: services, utilities, factories
  • Keep private variables at module scope

16.1 Understanding Vectors

Vectors represent points and directions in 3D space:

-- Creating vectors
local origin = Vector3.new(0, 0, 0)
local point = Vector3.new(10, 5, 3)
local direction = Vector3.new(1, 0, 0)  -- Unit vector

-- Vector operations
local a = Vector3.new(1, 2, 3)
local b = Vector3.new(4, 5, 6)

local sum = a + b           -- Addition
local diff = a - b          -- Subtraction
local scaled = a * 2        -- Scalar multiplication
local divided = a / 2       -- Scalar division

-- Vector properties
local magnitude = a.Magnitude  -- Length (3.74...)
local unit = a.Unit            -- Normalized direction

-- Distance between points
local distance = (a - b).Magnitude

-- Dot product (angle between vectors)
local dot = a:Dot(b)

-- Cross product (perpendicular vector)
local cross = a:Cross(b)

16.2 Understanding CFrame

CFrame (Coordinate Frame) combines position and orientation:

-- Creating CFrames
local cf1 = CFrame.new(10, 5, 0)  -- Position only
local cf2 = CFrame.new(Vector3.new(10, 5, 0))  -- From vector

-- Position and look-at
local cf3 = CFrame.lookAt(
    Vector3.new(0, 10, 0),    -- Position
    Vector3.new(0, 0, 0)      -- Look at this point
)

-- From position and orientation
local cf4 = CFrame.new(0, 5, 0) * CFrame.Angles(0, math.rad(45), 0)

-- CFrame properties
local position = cf1.Position
local lookVector = cf1.LookVector
local rightVector = cf1.RightVector
local upVector = cf1.UpVector

local part = Instance.new("Part")

part.Parent = workspace

-- Setting with CFrame (position + rotation)

-- Relative movement

part.CFrame = part.CFrame * CFrame.new(5, 0, 0)  -- Move right 5 (local)

-- Rotation

17.1 Understanding Events

Events signal when something happens in the game:

-- Always check if objects still exist

    if not part.Parent then return end
    if not hit.Parent then return end
    -- Process touch
end)

    if debounce then return end
    debounce = true
    
    -- Process touch
    print("Touched!")
    task.wait(1)  -- 1 second cooldown
    
    debounce = false
end)

-- Store connections for cleanup
local connections = {}

        connection:Disconnect()

    end)

    end)

function CurrencyService:getBalance(player)
    return playerBalances[player] or 0
end

function CurrencyService:add(player, amount)
    if amount <= 0 then return false end
    
    local current = self:getBalance(player)
    local newBalance = math.min(current + amount, MAX_BALANCE)
    
    playerBalances[player] = newBalance
    self:logTransaction(player, "add", amount)
    
    return true
end

function CurrencyService:subtract(player, amount)
    if amount <= 0 then return false end
    
    local current = self:getBalance(player)
    if current < amount then return false end
    
    playerBalances[player] = current - amount
    self:logTransaction(player, "subtract", amount)
    
    return true
end

-- Private function
function CurrencyService:logTransaction(player, type, amount)
    table.insert(transactionHistory, {
        player = player.UserId,
        type = type,
        amount = amount,
        time = os.time()
    })
end

return CurrencyService

22.2 Event Bus Pattern

-- EventBus.lua
local EventBus = {}

local events = {}

function EventBus:on(eventName, callback)
    if not events[eventName] then
        events[eventName] = {}
    end
    
    table.insert(events[eventName], callback)
    
    -- Return unsubscribe function
    return function()
        for i, cb in ipairs(events[eventName]) do
            if cb == callback then
                table.remove(events[eventName], i)
                break
            end
        end
    end
end

function EventBus:emit(eventName, ...)
    if not events[eventName] then return end
    
    for _, callback in ipairs(events[eventName]) do
        task.spawn(callback, ...)  -- Spawn in new thread
    end
end

return EventBus

-- Position validation
function validatePosition(player)

    if not rootPart then return end
    
    local currentPos = rootPart.Position
    local lastPos = lastPositions[player]
    
    if lastPos then

        local timeDelta = tick() - lastPositionTimes[player]

25.3 Rate Limiting

-- Prevent remote event spam
local rateLimits = {}
local RATE_LIMIT = 10  -- Max calls per second

function checkRateLimit(player, action)
    local key = player.UserId .. "_" .. action
    local now = tick()
    
    if not rateLimits[key] then
        rateLimits[key] = {count = 0, resetTime = now + 1}
    end
    
    local limit = rateLimits[key]
    
    if now > limit.resetTime then
        limit.count = 0
        limit.resetTime = now + 1
    end
    
    limit.count = limit.count + 1
    
    if limit.count > RATE_LIMIT then
        warn(player.Name .. " exceeded rate limit for " .. action)
        return false
    end
    
    return true
end

    if not checkRateLimit(player, "action") then
        return  -- Ignore the request
    end
    
    -- Process the request
end)

29.1 Tycoon System

-- TycoonSystem.lua
local TycoonSystem = {}

local playerTycoons = {}
local BASE_INCOME = 10
local INCOME_INTERVAL = 1

function TycoonSystem:init()
    local Players = game:GetService("Players")
    
    Players.PlayerAdded:Connect(function(player)
        self:createTycoon(player)
    end)
    
    Players.PlayerRemoving:Connect(function(player)
        playerTycoons[player] = nil
    end)
    
    -- Income loop
    task.spawn(function()
        while true do
            task.wait(INCOME_INTERVAL)
            self:distributeIncome()
        end
    end)
end

function TycoonSystem:createTycoon(player)
    playerTycoons[player] = {
        money = 0,
        totalEarned = 0,
        multipliers = {
            base = 1,
            upgrades = 1,
            gamepass = 1
        },
        upgrades = {},
        collectors = 1
    }
end

function TycoonSystem:distributeIncome()
    for player, tycoon in pairs(playerTycoons) do
        local income = BASE_INCOME * tycoon.collectors
        
        for _, multiplier in pairs(tycoon.multipliers) do
            income = income * multiplier
        end
        
        tycoon.money = tycoon.money + income
        tycoon.totalEarned = tycoon.totalEarned + income
        
        -- Update client
        self:updateClient(player)
    end
end

function TycoonSystem:purchaseUpgrade(player, upgradeId)
    local tycoon = playerTycoons[player]
    if not tycoon then return false end
    
    local upgrade = UpgradeDatabase[upgradeId]
    if not upgrade then return false end
    
    if tycoon.money < upgrade.cost then return false end
    if tycoon.upgrades[upgradeId] then return false end
    
    tycoon.money = tycoon.money - upgrade.cost
    tycoon.upgrades[upgradeId] = true
    
    -- Apply upgrade effect
    if upgrade.type == "multiplier" then
        tycoon.multipliers.upgrades = tycoon.multipliers.upgrades * upgrade.value
    elseif upgrade.type == "collector" then
        tycoon.collectors = tycoon.collectors + upgrade.value
    end
    
    return true
end

function TycoonSystem:updateClient(player)
    local tycoon = playerTycoons[player]
    if not tycoon then return end
    
    local event = game.ReplicatedStorage.Events.UpdateTycoon
    event:FireClient(player, {
        money = tycoon.money,
        totalEarned = tycoon.totalEarned,
        collectors = tycoon.collectors
    })
end

return TycoonSystem
